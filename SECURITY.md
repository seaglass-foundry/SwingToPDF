# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in swingtopdf, please report it responsibly by emailing [rich@seaglassfoundry.com](mailto:rich@seaglassfoundry.com). Do not open a public GitHub issue for security vulnerabilities.

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The version of swingtopdf you are using
- Any relevant environment details (Java version, OS)

## Response

We will acknowledge your report within 5 business days and aim to provide a fix or mitigation plan within 30 days, depending on severity.

## Scope

This policy applies to the swingtopdf library (`com.seaglassfoundry:swingtopdf`) and its source code hosted at [github.com/seaglass-foundry/swingtopdf](https://github.com/seaglass-foundry/swingtopdf).
